2) Web Sockets

Web Sockets provide a full-duplex communication channel over a single, long-lived connection between a client and a server.
This enables real-time data transfer between the two, facilitating applications that require continuous exchange of data,
such as chat applications, online gaming, live notifications, collaborative editing tools, and financial trading platforms.
Unlike traditional HTTP requests, Web Sockets allow for bi-directional communication, meaning both the client and the server
can send data at any time without waiting for a request.



3)Web Workers:

Web Workers are a mechanism that allows web applications to run scripts in background threads separate from the main execution thread of the web page.
This enables concurrent execution, preventing long-running scripts from blocking the main thread and thus improving the responsiveness and performance of web applications
Web Workers are commonly used for tasks such as
1) complex calculations,
2) data processing,
3) fetching data from APIs,
and performing other CPU-intensive operations without affecting the user interface's responsiveness. can send data at any time without waiting for a request.
